# 视觉 · 图像分类

## 1. 模块概述

- 主要功能：基于 ResNet50 的图像分类，输入一张图片，输出 Top-K 类别预测与置信度。支持 ImageNet 1000 类。
- 规格或特性：
  - 支持模型：ResNet50
  - 输入尺寸：`[1, 3, 224, 224]`
  - 量化类型：int8
  - 输出：Top-K 类别 ID + 置信度
  - 推理后端：ONNX Runtime + SpaceMITExecutionProvider
  - 接口形态：C++（`vision_service.h`）、Python（`ModelFactory`）
- 相关目录结构：

```
examples/resnet/
├── config/resnet50.yaml   # 配置文件
├── cpp/resnet.cpp         # C++ 示例
├── python/resnet.py       # Python 示例
└── scripts/               # 模型下载脚本
src/deploy/resnet/         # 部署实现（C++ / Python）
assets/labels/imagenet.txt # ImageNet 1000 类标签
```

## 2. 环境准备

### 前置条件

- 运行环境：Bianbu Linux（K3 平台）
- 编译环境：CMake 3.10+，C++17；Python 3.12+（可选）
- C++ 依赖：OpenCV 4、spacemit-ort、yaml-cpp、Eigen3

- 硬件：SpacemiT K3 开发板

## 4. 应用开发

本章面向应用开发者，说明如何在自己的 C++ 或 Python 应用中集成图像分类组件。完整接口定义以 `include/vision_service.h` 和 `src/core/python/vision_model_factory.py` 为准；本节介绍常用公开接口和典型调用方式。

### 4.1 接口说明

图像分类组件的核心入口是 `VisionService`（C++）和 `ModelFactory`（Python）。应用侧通过这些接口加载 ResNet 模型，并发起图像分类请求。

#### 4.1.1 常用数据结构

| 类型 | 说明 |
| --- | --- |
| VisionServiceResult | 分类结果，包含类别 ID（label）、置信度（score）。 |
| VisionServiceConfig | 配置参数，包含模型路径、输入尺寸、线程数、推理后端等。 |

#### 4.1.2 服务初始化

**C++ 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| VisionService::Create | 从 YAML 配置文件创建分类服务实例 | config_path：YAML 配置文件路径 | VisionService 智能指针 |
| VisionService::LastCreateError | 获取最近一次创建失败的错误信息 | 无 | 错误描述字符串 |

**Python 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| ModelFactory.create | 从 YAML 配置文件创建分类器实例 | config_path：YAML 配置文件路径 | 分类器对象 |

#### 4.1.3 图像分类

**C++ 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| InferImage | 对单张图像进行分类 | image_path：图像文件路径；results：输出结果向量 | bool（成功/失败） |
| InferImage | 对 cv::Mat 图像进行分类 | image：OpenCV Mat 对象；results：输出结果向量 | bool（成功/失败） |
| LastError | 获取最近一次推理的错误信息 | 无 | 错误描述字符串 |

**Python 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| predict_top_k | 对图像进行分类并返回 Top-K 结果 | image_path：图像文件路径或 numpy 数组；k：返回前 k 个结果 | 分类结果列表（类别名、置信度） |

#### 4.1.4 性能监控

**C++ 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| SetTimingOptions | 启用/禁用性能计时 | enable：是否启用 | void |
| GetTimingInfo | 获取最近一次推理的各阶段耗时 | 无 | TimingInfo 结构体（预处理、推理、后处理耗时） |

### 4.2 典型调用流程

#### 4.2.1 C++ 单图分类

```cpp
#include "vision_service.h"
#include <opencv2/opencv.hpp>
#include <iostream>

int main() {
    // 1. 创建服务
    auto service = VisionService::Create("examples/resnet/config/resnet50.yaml");
    if (!service) {
        std::cerr << "Failed to create service: " 
                  << VisionService::LastCreateError() << std::endl;
        return -1;
    }

    // 2. 启用性能计时（可选）
    service->SetTimingOptions(true);

    // 3. 执行推理
    std::vector<VisionServiceResult> results;
    if (!service->InferImage("test.jpg", &results)) {
        std::cerr << "Inference failed: " << service->LastError() << std::endl;
        return -1;
    }

    // 4. 处理结果（按置信度排序）
    std::sort(results.begin(), results.end(), 
              [](const auto& a, const auto& b) { return a.score > b.score; });
    
    std::cout << "Classification results (Top-5):" << std::endl;
    for (size_t i = 0; i < std::min(size_t(5), results.size()); ++i) {
        std::cout << "  " << (i+1) << ". Class " << results[i].label 
                  << ", Confidence: " << results[i].score << std::endl;
    }

    // 5. 查看性能指标（可选）
    auto timing = service->GetTimingInfo();
    std::cout << "Preprocess: " << timing.preprocess_ms << " ms" << std::endl;
    std::cout << "Inference: " << timing.inference_ms << " ms" << std::endl;
    std::cout << "Postprocess: " << timing.postprocess_ms << " ms" << std::endl;

    return 0;
}
```

#### 4.2.2 C++ 批量图像分类

```cpp
#include "vision_service.h"
#include <opencv2/opencv.hpp>
#include <filesystem>

int main() {
    auto service = VisionService::Create("examples/resnet/config/resnet50.yaml");
    
    // 遍历目录中的所有图像
    for (const auto& entry : std::filesystem::directory_iterator("images/")) {
        if (entry.path().extension() == ".jpg") {
            std::vector<VisionServiceResult> results;
            service->InferImage(entry.path().string(), &results);
            
            // 输出 Top-1 结果
            if (!results.empty()) {
                auto top1 = *std::max_element(results.begin(), results.end(),
                    [](const auto& a, const auto& b) { return a.score < b.score; });
                std::cout << entry.path().filename() << ": Class " 
                          << top1.label << " (" << top1.score << ")" << std::endl;
            }
        }
    }

    return 0;
}
```

#### 4.2.3 Python 单图分类

```python
from src.core.python.vision_model_factory import ModelFactory

# 1. 创建分类器
classifier = ModelFactory.create("examples/resnet/config/resnet50.yaml")

# 2. 执行推理（返回 Top-5 结果）
results = classifier.predict_top_k("test.jpg", k=5)

# 3. 处理结果
print("Classification results (Top-5):")
for i, (class_name, confidence) in enumerate(results, 1):
    print(f"  {i}. {class_name} (confidence: {confidence:.4f})")
```

#### 4.2.4 Python 批量图像分类

```python
from src.core.python.vision_model_factory import ModelFactory
import os
import glob

classifier = ModelFactory.create("examples/resnet/config/resnet50.yaml")

# 批量处理目录中的所有图像
for image_path in glob.glob("images/*.jpg"):
    results = classifier.predict_top_k(image_path, k=1)
    if results:
        class_name, confidence = results[0]
        print(f"{os.path.basename(image_path)}: {class_name} ({confidence:.4f})")
```

### 4.3 配置说明

YAML 配置文件是模型加载和推理参数的核心，以下是完整配置项说明：

```yaml
# 模型文件路径（ResNet50 ImageNet 分类模型）
model_path: ~/.cache/models/vision/resnet/resnet50.q.onnx

# 测试图像路径（用于示例程序）
test_image: ~/.cache/assets/image/001_test.jpg

# 类别标签文件路径（ImageNet 1000 类）
label_file_path: assets/labels/imagenet.txt

# 模型输入尺寸 [height, width]（ResNet 标准输入为 224×224）
image_size: [224, 224]

# 部署类名（用于 Python ModelFactory）
class: deploy.resnet.ResNetClassifier

# 推理参数
default_params:
  # 推理线程数（K3 平台建议 4）
  num_threads: 4
  
  # 推理后端（优先使用 SpaceMITExecutionProvider）
  providers:
    - SpaceMITExecutionProvider
    - CPUExecutionProvider  # 备用后端
```

**参数调优建议**：

- **image_size**：ResNet 标准输入为 [224, 224]，不建议修改。
- **num_threads**：K3 平台建议设为 4，过高可能导致性能下降。
- **providers**：优先使用 SpaceMITExecutionProvider 以获得最佳性能。
- **label_file_path**：确保指向正确的 ImageNet 标签文件，否则只能输出类别 ID。

### 4.4 性能监控

通过启用性能计时，可以分析推理各阶段的耗时，用于性能优化和瓶颈定位。

**C++ 示例**：

```cpp
service->SetTimingOptions(true);
service->InferImage("test.jpg", &results);

auto timing = service->GetTimingInfo();
std::cout << "Preprocess: " << timing.preprocess_ms << " ms" << std::endl;
std::cout << "Inference: " << timing.inference_ms << " ms" << std::endl;
std::cout << "Postprocess: " << timing.postprocess_ms << " ms" << std::endl;
std::cout << "Total: " << (timing.preprocess_ms + timing.inference_ms + timing.postprocess_ms) << " ms" << std::endl;
```

**性能优化建议**：

- 预处理耗时高：图像分类的预处理相对简单（缩放 + 归一化），耗时应较低。
- 推理耗时高：确认使用 SpaceMITExecutionProvider，检查线程数设置。
- 后处理耗时高：分类任务的后处理仅为 softmax + 排序，耗时应极低。

**参考 demo 路径**：`examples/resnet/`

## 5. 调试指南

- 启用计时：通过 `SetTimingOptions` 查看各阶段耗时
- 分类结果不准确：确认输入图片已正确预处理（缩放至 224×224）
- 标签不匹配：确认 `label_file_path` 指向正确的 `imagenet.txt`

## 6. 常见问题

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| `Model file not found` | 模型未下载 | 执行 `bash examples/resnet/scripts/download_models.sh` |
| 分类结果全为低置信度 | 输入图片不在 ImageNet 类别范围内 | 换用 ImageNet 类别内的测试图片 |
| 输出类别 ID 而非名称 | 标签文件缺失 | 确认 `assets/labels/imagenet.txt` 存在 |
