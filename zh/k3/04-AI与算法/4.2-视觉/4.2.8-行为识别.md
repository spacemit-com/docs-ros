# 视觉 · 行为识别

## 1. 模块概述

- 主要功能：基于 STGCN（Spatial-Temporal Graph Convolutional Network）的骨架行为识别，输入连续 30 帧的人体关键点序列，输出动作类别概率。支持 7 类动作识别，包含跌倒检测（Fall Down）。通常与 YOLOv8-Pose 配合使用：先检测人体关键点，再将关键点序列送入 STGCN 进行动作分类。
- 规格或特性：
  - 支持模型：STGCN（TSSTG）
  - 输入：30 帧骨架序列（17 个 COCO 关键点 × 30 帧）
  - 数据类型：fp32
  - 输出：7 类动作概率
  - 推理后端：ONNX Runtime + CPUExecutionProvider
  - 接口形态：C++（`vision_service.h`）、Python（`ModelFactory`）
- 相关目录结构：

```
src/deploy/stgcn/
├── cpp/stgcn_action_recognizer.h      # C++ 头文件
├── cpp/stgcn_action_recognizer.cpp    # C++ 实现
└── python/stgcn_action_recognizer.py  # Python 实现
applications/fall_detection/
├── config/fall_detection.yaml         # 跌倒检测应用配置
├── config/stgcn_action.yaml           # STGCN 模型配置
├── cpp/example_fall_detection.cpp     # C++ 跌倒检测示例
├── python/example_fall_detection.py   # Python 跌倒检测示例
└── scripts/download_models.sh         # 模型下载脚本
```

## 4. 应用开发

本章面向应用开发者，说明如何在自己的 C++ 或 Python 应用中集成行为识别组件。完整接口定义以 `include/vision_service.h` 和 `src/core/python/vision_model_factory.py` 为准；本节介绍常用公开接口和典型调用方式。

### 4.1 接口说明

行为识别组件基于 STGCN（时空图卷积网络）模型，需要输入连续 30 帧的人体关键点序列进行动作分类。应用侧通常先使用姿态估计模型提取关键点，再送入 STGCN 进行行为识别。

#### 4.1.1 常用数据结构

| 类型 | 说明 |
| --- | --- |
| VisionServiceResult | 行为识别结果，包含动作类别 ID（label）、置信度（score）。 |
| VisionServiceConfig | 配置参数，包含模型路径、线程数、推理后端等。 |

#### 4.1.2 服务初始化

**C++ 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| VisionService::Create | 从 YAML 配置文件创建 STGCN 服务实例 | config_path：YAML 配置文件路径 | VisionService 智能指针 |
| GetSequenceClassNames | 获取所有动作类别名称 | 无 | 类别名称向量 |
| GetFallDownClassIndex | 获取跌倒动作的类别索引 | 无 | 类别索引（通常为 6） |

**Python 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| ModelFactory.create | 从 YAML 配置文件创建 STGCN 实例 | config_path：YAML 配置文件路径 | STGCN 对象 |

#### 4.1.3 行为识别

**C++ 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| InferSequence | 对 30 帧关键点序列进行动作分类 | pts：关键点数组；width：图像宽度；height：图像高度；scores：输出分数向量 | bool（成功/失败） |

**Python 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| predict | 对关键点序列进行动作分类 | keypoint_sequence：30 帧关键点数据 | 动作分类分数 |

#### 4.1.4 性能监控

**C++ 接口**

| 接口 | 说明 | 参数 | 返回值 |
| --- | --- | --- | --- |
| SetTimingOptions | 启用/禁用性能计时 | enable：是否启用 | void |
| GetTimingInfo | 获取最近一次推理的各阶段耗时 | 无 | TimingInfo 结构体（包含 sequence_ms 字段） |

### 4.2 典型调用流程

#### 4.2.1 C++ 跌倒检测应用

```cpp
#include "vision_service.h"
#include <opencv2/opencv.hpp>
#include <deque>

int main() {
    // 1. 创建姿态估计服务
    auto pose_detector = VisionService::Create("examples/yolov8_pose/config/yolov8_pose.yaml");
    
    // 2. 创建 STGCN 服务
    auto stgcn = VisionService::Create("applications/fall_detection/config/stgcn_action.yaml");
    
    // 3. 获取动作类别信息
    auto class_names = stgcn->GetSequenceClassNames();
    int fall_idx = stgcn->GetFallDownClassIndex();
    
    // 4. 打开视频流
    cv::VideoCapture cap("video.mp4");
    cv::Mat frame;
    
    // 5. 关键点序列缓冲区（30 帧）
    std::deque<std::vector<cv::Point2f>> keypoint_buffer;
    
    while (cap.read(frame)) {
        // 6. 姿态估计
        std::vector<VisionServiceResult> pose_results;
        pose_detector->InferImage(frame, &pose_results);
        
        if (!pose_results.empty()) {
            // 提取第一个人的关键点
            std::vector<cv::Point2f> keypoints;
            for (const auto& kp : pose_results[0].keypoints) {
                keypoints.push_back(cv::Point2f(kp.x, kp.y));
            }
            
            // 7. 更新关键点缓冲区
            keypoint_buffer.push_back(keypoints);
            if (keypoint_buffer.size() > 30) {
                keypoint_buffer.pop_front();
            }
            
            // 8. 当累积 30 帧时进行行为识别
            if (keypoint_buffer.size() == 30) {
                // 将关键点序列转换为 float 数组
                std::vector<float> pts;
                for (const auto& frame_kps : keypoint_buffer) {
                    for (const auto& kp : frame_kps) {
                        pts.push_back(kp.x);
                        pts.push_back(kp.y);
                    }
                }
                
                // 9. STGCN 推理
                std::vector<float> scores;
                stgcn->InferSequence(pts.data(), frame.cols, frame.rows, &scores);
                
                // 10. 判断是否跌倒
                if (scores[fall_idx] > 0.5) {
                    std::cout << "Fall detected! Score: " << scores[fall_idx] << std::endl;
                }
            }
        }
        
        cv::imshow("Fall Detection", frame);
        if (cv::waitKey(1) == 'q') break;
    }
    
    return 0;
}
```

#### 4.2.2 Python 跌倒检测应用

```python
from src.core.python.vision_model_factory import ModelFactory
import cv2
from collections import deque

# 1. 创建姿态估计器和 STGCN
pose_detector = ModelFactory.create("examples/yolov8_pose/config/yolov8_pose.yaml")
stgcn = ModelFactory.create("applications/fall_detection/config/stgcn_action.yaml")

# 2. 打开视频流
cap = cv2.VideoCapture("video.mp4")

# 3. 关键点序列缓冲区（30 帧）
keypoint_buffer = deque(maxlen=30)

while True:
    ret, frame = cap.read()
    if not ret:
        break
    
    # 4. 姿态估计
    pose_results = pose_detector.detect(frame)
    
    if pose_results:
        # 提取第一个人的关键点
        keypoints = pose_results[0]['keypoints']
        keypoint_buffer.append(keypoints)
        
        # 5. 当累积 30 帧时进行行为识别
        if len(keypoint_buffer) == 30:
            # 将关键点序列转换为 numpy 数组
            keypoint_sequence = list(keypoint_buffer)
            
            # 6. STGCN 推理
            scores = stgcn.predict(keypoint_sequence)
            
            # 7. 判断是否跌倒（类别 6）
            if scores[6] > 0.5:
                print(f"Fall detected! Score: {scores[6]:.4f}")
    
    cv2.imshow("Fall Detection", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

### 4.3 配置说明

#### 4.3.1 STGCN 模型配置

```yaml
# STGCN 模型文件路径（fp32 格式）
model_path: ~/.cache/models/vision/stgcn/stgcn.fp32.onnx

# 部署类名（用于 Python ModelFactory）
class: deploy.stgcn.StgcnActionRecognizer

# 推理参数
default_params:
  # 推理线程数（K3 平台建议 4）
  num_threads: 4
  
  # 是否延迟加载模型
  lazy_load: false
  
  # 推理后端（STGCN 使用 CPU，因为是 fp32 格式）
  providers:
    - CPUExecutionProvider
```

**参数调优建议**：

- **stgcn_wait_frames**：控制开始推理的延迟，增大可减少初始误报。
- **stgcn_smooth_window**：平滑窗口大小，增大可减少误报但会增加延迟。
- **kp_threshold**：关键点可见度阈值，过低会引入噪声关键点。
- **providers**：STGCN 使用 CPUExecutionProvider，因为模型为 fp32 格式。

### 4.4 性能监控

通过启用性能计时，可以分析推理各阶段的耗时，用于性能优化和瓶颈定位。

**C++ 示例**：

```cpp
stgcn->SetTimingOptions(true);
stgcn->InferSequence(pts, width, height, &scores);

auto timing = stgcn->GetTimingInfo();
std::cout << "Sequence inference: " << timing.sequence_ms << " ms" << std::endl;
```

**性能优化建议**：

- STGCN 推理耗时相对较低（单次推理 < 50ms），整体流水线性能主要受前端姿态检测影响。
- 姿态检测是瓶颈：优化姿态检测模型（使用 yolov8n-pose 而非 yolov8m-pose）。
- 减少推理频率：不必每帧都进行 STGCN 推理，可每隔几帧推理一次。

**参考 demo 路径**：`applications/fall_detection/`

## 5. 调试指南

- STGCN 使用 `CPUExecutionProvider`（非 SpaceMIT），因为模型为 fp32 格式
- 启用计时：通过 `SetTimingOptions` 查看 `sequence_ms` 字段
- 跌倒误报：调整 `stgcn_smooth_window`（增大可减少误报）和 `kp_threshold`
- 确认姿态检测正常工作后再排查 STGCN 问题

## 6. 常见问题

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| `Model file not found` | STGCN 或 Pose 模型未下载 | 分别执行两个模型的下载脚本 |
| 动作分类不准确 | 关键点检测质量差 | 确认 YOLOv8-Pose 检测正常，调整 `kp_threshold` |
| 跌倒检测延迟高 | 需要累积 30 帧 | 正常现象，STGCN 需要 30 帧序列输入 |
| 频繁误报跌倒 | 平滑窗口过小 | 增大 `stgcn_smooth_window`（如 7 或 9） |
| qt.qpa.xcb: could not connect to display | 没有接入显示器 | 接入显示器 |
